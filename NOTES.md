# Notes
The system uses XOR-based Forward Error Correction (FEC) rather than ARQ to respect the strict playout deadlines without requiring round-trip network times. To stay well under the 2.0x bandwidth cap, the sender only injects parity data (XOR of the previous two payloads) on even sequence numbers, resulting in a ~1.5x overhead. The receiver maintains a ring buffer and continuously sweeps it to mathematically reconstruct missing packets using the parity data. We also compress the 32-bit sequence number to an 8-bit integer on the wire to save a few bytes. The system should be graded at a delay of 105 ms. It will break if consecutive packets are dropped across an entire parity window (e.g., losing frames i, i+1, and i+2 in a burst), making mathematical recovery impossible.
